Python (along with many other programming languages) represents floating point numbers that are not always perfectly precise. This can cause small "rounding" errors to happen.

For example, notice this odd error:

>>> 0.1 + 0.1
0.2
>>> 0.1 + 0.1 + 0.1
0.30000000000000004
>>>

Most of the time, these small errors don't matter. But it is something to be aware of.

Here's a good technical discussion about it: What Every Computer Scientist Should Know About Floating-Point Arithmetic
